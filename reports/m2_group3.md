## Build & Run Instructions

Full details: [DEPLOYMENT.md](https://github.com/SWE-574/SWE-574-3/blob/dev/DEPLOYMENT.md)

### Prerequisites

- **Docker** & **Docker Compose** (v2)
- **Git**
- A copy of the `.env` file (generate with `make env`, or copy from `.env.example` and fill in values)

### Quick Start — Docker (recommended)

```bash
git clone https://github.com/SWE-574/SWE-574-3.git
cd SWE-574-3
make env          # interactive .env generator
make docker-up    # starts full stack (backend, frontend, DB, Redis, MinIO, Nginx) on port 80
```

Open [http://localhost](http://localhost) in your browser.

To stop: `make docker-down`

### Local Development (native backend & frontend)

```bash
make env          # generate .env
make setup        # install Python/Node dependencies, run migrations, seed data
make dev          # starts backend (port 8000), frontend (port 5173), and infra containers (DB, Redis, MinIO)
```

- Frontend: [http://localhost:5173](http://localhost:5173)
- Backend API: [http://localhost:8000](http://localhost:8000)

### Production Deployment

```bash
make env          # set DEBUG=False, ALLOWED_HOSTS, DOMAIN, LETSENCRYPT_DIR
make prod-up      # multi-stage build, Daphne ASGI, HTTPS via Nginx
```

TLS certificates are managed via Let's Encrypt / Certbot (see DEPLOYMENT.md for details).

### Key Environment Variables

| Variable | Purpose |
|---|---|
| `DB_NAME`, `DB_USER`, `DB_PASSWORD` | PostgreSQL / PostGIS credentials |
| `SECRET_KEY` | Django secret key |
| `VITE_API_URL` | Backend URL for the frontend |
| `VITE_MAPBOX_TOKEN` | Mapbox token for map features |
| `MINIO_ACCESS_KEY`, `MINIO_SECRET_KEY` | S3-compatible object storage |
| `EXPO_PUBLIC_API_URL` | Backend URL for the mobile app |

## Mobile Application

The mobile app is built with **React Native (Expo)** and deployed via **Expo Application Services (EAS)**.

### Running on a Device (pre-built APK)

1. Download the APK from the [0.2.0-alpha release assets](https://github.com/SWE-574/SWE-574-3/releases/tag/customer-milestone-2)
2. Install on any Android device (enable "Install from unknown sources" if prompted)
3. The APK is pre-configured to connect to the production backend at `apiary.selmangunes.com`

### Local Mobile Development

```bash
cd mobile-client
npm install
```

| Command | Description |
|---|---|
| `npm start` | Start Expo dev server (scan QR with Expo Go) |
| `npm run android` | Build and run on Android emulator |
| `npm run ios` | Build and run on iOS simulator (macOS only) |
| `npm run start:local:android` | Android emulator pointed at local backend |
| `npm run start:local:ios` | iOS simulator pointed at local backend |
| `npm run start:local:device` | Physical device via LAN pointed at local backend |

> **Note:** For local development with a physical device, set `EXPO_PUBLIC_API_URL` in your root `.env` to your machine's LAN IP (e.g., `http://192.168.1.x:8000`).

### Building Release APK / IPA

```bash
cd mobile-client
eas build -p android --profile production   # produces APK
eas build -p ios --profile production        # produces IPA (requires Apple Developer account)
```

Build profiles are defined in `mobile-client/eas.json` with three tiers: `development`, `preview`, and `production`.
