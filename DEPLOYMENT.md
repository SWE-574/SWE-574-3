# Deployment Guide

Three ways to run The Hive — all powered by **Docker Compose v2** (`docker compose`).

Backend and infrastructure share a **single `.env` file** at the project root. The frontend (Vite) also reads this root `.env` via `envDir` in `vite.config.ts`, so `VITE_*` variables work from the same file. Run `make env` to generate it interactively, or copy `.env.example` and edit manually.

---

## 1. Local Development (recommended)

Infrastructure (PostGIS, Redis, MinIO) runs in Docker; backend and frontend run natively for fast reload.

**Prerequisites:** Docker Desktop, Python 3.11+, Node.js 20+  
**macOS (Apple Silicon):** `brew install gdal geos`

```bash
make env          # interactive .env generator (one-time)
make setup        # venv + deps + infra + migrate
make dev          # start backend :8000 + frontend :5173  (Ctrl+C to stop)
```

Want demo accounts? Use `make setup-demo` instead of `make setup`.

| URL | Description |
|-----|-------------|
| http://localhost:5173 | Frontend (Vite dev server) |
| http://localhost:8000/api/ | REST API |
| http://localhost:8000/api/docs/ | Swagger UI |
| http://localhost:9001 | MinIO console |

**Demo accounts** (password `demo123`): `elif@demo.com`, `cem@demo.com`, `moderator@demo.com` (admin).

> **Port conflicts:** If 5432 or 6379 are already in use, override in `.env`:
> ```dotenv
> DB_PORT=55432
> REDIS_PORT=56379
> ```

## 2. Docker Dev (full stack)

Everything runs in containers behind Nginx on port 80. Uses `docker-compose.yml`.

```bash
make env              # if .env does not exist yet
make docker-up        # start all containers
make docker-demo      # start + seed demo data
```

| URL | Description |
|-----|-------------|
| http://localhost | Frontend |
| http://localhost/api/ | REST API |
| http://localhost/api/docs/ | Swagger UI |

Other useful commands:

```bash
make docker-logs      # tail logs
make docker-build     # rebuild images
make docker-down      # stop containers
make docker-reset     # stop + delete volumes (full wipe)
```

## 3. Production

Uses `docker-compose.prod.yml` with multi-stage frontend build, Daphne ASGI, and HTTPS via Nginx.

**Requirements:** Linux server with Docker Engine, ports 80 + 443 open.

### Quick start

```bash
git clone <repo-url> /opt/thehive && cd /opt/thehive
make env              # fill in production values when prompted
make prod-up          # start production stack
```

Or seed demo data on first deploy:

```bash
make prod-demo
```

### Required `.env` values for production

```dotenv
DB_PASSWORD=<strong-password>
SECRET_KEY=<auto-generated by make env>
DEBUG=False
ALLOWED_HOSTS=yourdomain.com
CORS_ALLOWED_ORIGINS=https://yourdomain.com
FRONTEND_URL=https://yourdomain.com
DOMAIN=yourdomain.com
```

### TLS with Let's Encrypt

```bash
# 1. Install certbot
apt install certbot -y

# 2. Stop nginx to free port 80 for the ACME challenge
make prod-down

# 3. Get certificate (point DNS A record to your server first)
certbot certonly --standalone -d yourdomain.com

# 4. Set in .env
#    LETSENCRYPT_DIR=/etc/letsencrypt

# 5. Start everything
make prod-up
```

**Auto-renewal** (add via `crontab -e`):

```cron
0 3 * * * cd /opt/thehive && make prod-down && certbot renew --quiet && make prod-up
```

### Updating

```bash
git pull
make prod-build
make prod-up
docker compose -f docker-compose.prod.yml exec backend python manage.py migrate
```

## Environment Variables

All config lives in a single `.env` at the project root. Run `make env` to generate it interactively.

| Variable | Default | Required in prod | Description |
|----------|---------|:---:|-------------|
| `DB_NAME` | `the_hive_db` | | PostgreSQL database name |
| `DB_USER` | `postgres` | | Database user |
| `DB_PASSWORD` | `postgres123` | **yes** | Use a strong password in production |
| `DB_HOST` | `localhost` | | Set automatically by Docker Compose |
| `DB_PORT` | `5432` | | Host port for PostGIS |
| `REDIS_HOST` | `localhost` | | Set automatically by Docker Compose |
| `REDIS_PORT` | `6379` | | Host port for Redis |
| `MINIO_ENDPOINT` | `localhost:9000` | | MinIO S3 API endpoint |
| `MINIO_ACCESS_KEY` | `minioadmin` | | MinIO access key |
| `MINIO_SECRET_KEY` | `minioadmin123` | | MinIO secret key |
| `SECRET_KEY` | — | **yes** | Django secret key (auto-generated by `make env`) |
| `DEBUG` | `True` | **yes** | Must be `False` in production |
| `ALLOWED_HOSTS` | `localhost,127.0.0.1` | **yes** | Comma-separated hostnames |
| `CORS_ALLOWED_ORIGINS` | `http://localhost:5173` | **yes** | Comma-separated origins |
| `FRONTEND_URL` | `http://localhost:5173` | **yes** | Used in email links |
| `RESEND_API_KEY` | — | | Email delivery; leave blank to disable emails |
| `RESEND_FROM_EMAIL` | `onboarding@resend.dev` | | Sender address (works on free tier) |
| `VITE_MAPBOX_TOKEN` | — | **yes** | Mapbox public token for the map |
| `DOMAIN` | — | **yes** | Nginx server name, e.g. `yourdomain.com` |
| `LETSENCRYPT_DIR` | — | **yes** | Path to Let's Encrypt certs |

### Mapbox token

The interactive map requires a Mapbox GL JS token.

1. Sign up at [mapbox.com](https://account.mapbox.com/) (free).
2. Copy your default public token (starts with `pk.`).
3. Paste it into `.env` as `VITE_MAPBOX_TOKEN`.
4. In production, restrict the token to your domain under **Access Tokens → URL restrictions**.

Free tier: 50,000 map loads/month.

### Resend (email)

Used for email verification and password resets.

1. Sign up at [resend.com](https://resend.com) (free tier: 100 emails/day).
2. Get your API key at [resend.com/api-keys](https://resend.com/api-keys).
3. Paste it into `.env` as `RESEND_API_KEY`.

Without a key, email features are silently disabled — the app still works.

## Make Targets

```
make env              # interactive .env generator (re-run to overwrite)

make setup            # local: venv + deps + infra + migrate
make setup-demo       # local: setup + demo data
make dev              # local: start backend + frontend
make stop             # local: stop infra containers
make reset            # local: stop + delete volumes

make infra-up         # start infra only (db, redis, minio)
make infra-down       # stop infra
make infra-demo       # infra + migrate + demo data

make docker-up        # Docker dev: start full stack
make docker-demo      # Docker dev: start + demo data
make docker-down      # Docker dev: stop
make docker-logs      # Docker dev: tail logs
make docker-build     # Docker dev: rebuild images
make docker-reset     # Docker dev: stop + delete volumes

make prod-up          # production: start
make prod-demo        # production: start + demo data
make prod-down        # production: stop
make prod-logs        # production: tail logs
make prod-build       # production: rebuild images
make prod-reset       # production: stop + delete volumes

make test             # run all native tests
make test-docker      # run backend tests in Docker
make coverage         # generate coverage reports
make clean            # remove caches and build artifacts
```
